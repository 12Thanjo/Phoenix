CMD

CMD allows for formatted logging to the console, with colors and time-stamps. Thanks to use of macros, CMD only logs in debug mode and not in release mode (saving performance). Logs from internal Phoenix are denoted as such.

Example:

// set log level to warning or higher
ph_log_level(PH_LOG_WARNING);

int num = 12;

// this will print
ph_warning("number: " << num);

// this will not print
ph_info("number: " << num);



// set log level to trace or higher
ph_log_level(PH_LOG_TRACE);

// this will print
ph_info("number: " << num);




// Creating a test class to showcase an example implementation of custom printing
class TestClass{
private:
int num;

public:
TestClass(int p_num) : num(p_num){};
~TestClass(){};
void print(){
std::cout << "[TestClass num: " << num << "]";
}

};


// overloads operator to allow for printing of TestClass instances
static std::ostream& operator << (std::ostream& os, TestClass& test){
test.print();
return os;
}


// create an instance of TestClass with a number value of 12
TestClass test(12);


// print thie instance of TestClass
ph_info("Test Print: " << test);

The cout type shown down the page is used where std::cout like syntax can be used to form messages.

Functions:

ph_log_level(int level)

[MACRO] Set the logging level. Options are:

  • PH_LOG_TRACE
  • PH_LOG_INFO
  • PH_LOG_WARNING
  • PH_LOG_ERROR
  • PH_LOG_FATAL

ph_assert(bool test, cout message)

[MACRO] Send an error message if a condition isn't true


Logging:

ph_trace(cout message)

[MACRO] log to the console at a trace level

ph_info(cout message)

[MACRO] log to the console at a info level

ph_warning(cout message)

[MACRO] log to the console at a warning level

ph_error(cout message)

[MACRO] log to the console at a error level

ph_fatal(cout message)

[MACRO] log to the console at a fatal level
